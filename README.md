# Shoreline Evolution & Flood Depth Modeling

Automated workflow for simulating coastal flood extent and depth under climate change scenarios using GRASS GIS and high-resolution DTM analysis.

## 📋 Overview

This repository contains two integrated scripts for modeling flood dynamics under extreme sea level (ESL) conditions:

1. **`flood_modeling_workflow.sh`** – GRASS GIS-based hydrodynamic modeling using least-cost path analysis (r.cost)
2. **`flood_visualization.py`** – Publication-quality static map generation with depth classification

The workflow processes multiple climate scenarios (SSP245, SSP585), return periods (10yr, 50yr, 100yr), and future projections (2020–2080) to assess coastal flood risk.

## 🎯 Key Features

- **Multi-scenario modeling**: SSP245 & SSP585 IPCC climate scenarios
- **Temporal coverage**: 2020, 2040, 2060, 2080 projections
- **Return periods**: 10-year, 50-year, 100-year extreme sea levels
- **Spatial analysis**: 
  - DTM-based flood propagation
  - Slope and roughness-weighted friction factors
  - Cost-distance damping for realistic depth attenuation
- **Dual depth calculation**: 
  - Raw depth (ESL – DTM elevation)
  - Advanced depth (cost-normalized damping factor applied)
- **Batch processing**: Automated raster generation & export
- **Visualization**: Georeferenced flood depth maps with location overlays

  - OSM background raster (for context)

## 🚀 Usage

### Step 1: Prepare GRASS GIS Environment
```bash
# In GRASS GIS terminal, switch to your working mapset:
g.mapset mapset=your_mapset_name

# Ensure all input rasters/vectors are imported in this mapset
g.list type=raster
g.list type=vector
```

### Step 2: Configure & Run Flood Modeling

Edit `flood_modeling_workflow.sh` and update these paths:
```bash
DTM="dtm"                                # Your DTM raster name
ROUGHNESS="roughness_papaio"             # Your roughness raster
SHORELINE_RAST="shoreline_rast"          # Your shoreline starting raster
LAND_MASK="landmask"                     # Your land mask vector/raster
EXPORT_DIR="/path/to/your/output"       # Output directory
BASE_DIR="/path/to/esl_projections"     # ESL water level data directory
```

Run the script in GRASS GIS terminal:
```bash
bash flood_modeling_workflow.sh
```

**Processing time**: ~2–5 minutes per scenario (varies with resolution & region size)

### Step 3: Generate Visualizations

Edit `flood_visualization.py` and update paths:
```python
OSM_BACKGROUND = "/path/to/osm_background.tif"
LECCE_ROADS = "/path/to/roads.shp"
FLOOD_DIR = "/path/to/grass_outputs"      # Same as EXPORT_DIR above
OUTPUT_DIR = "/path/to/map_outputs"
```

Run visualization script:
```bash
python3 flood_visualization.py
```

**Output**: High-resolution PNG maps saved to `OUTPUT_DIR/with_damping/projection_{scenario}/rl{period}/`

## 🌊 Data Inputs & Processing Steps

### ESL Water Level Data
- **Format**: GeoPackage (`.gpkg`) with point geometries and `value` column (elevation in m)
- **Directory structure**:
water_levels/

├── projectionssp245/

│   ├── rl10/rl_10yr_2020.gpkg

│   ├── rl50/rl_50yr_2020.gpkg

│   └── rl100/rl_100yr_2020.gpkg

└── projectionssp585/

└── ...

### Processing Workflow

1. **Vector → Raster**: Points converted to raster using `v.to.rast`
2. **Spatial Interpolation**: `r.grow.distance` propagates nearest coastal point value to all cells
3. **Flood Detection**: `r.mapcalc` identifies cells where DTM < ESL
4. **Slope & Roughness**: Terrain derivatives (slope, aspect, curvature) computed via `r.slope.aspect`
5. **Friction Layer**: Cost surface = slope_factor × roughness_factor
6. **Flood Propagation**: `r.cost` computes least-cost distance from shoreline
7. **Depth Calculation**:
   - **Raw depth** = ESL raster – DTM
   - **Advanced depth** = raw_depth × (1 – normalized_cost)
   - Damping factor reduces depth for cells far from shoreline (realistic attenuation)
8. **Export**: GeoTIFF files with georeference preserved

## ⚙️ Technical Notes

- **Coordinate System**: EPSG:7792 (local UTM) internally; EPSG:4326 (WGS84) for visualization
- **Null handling**: `nan_to_num` converts no-data to 0 in Python; GRASS masks control spatial extent
- **Memory**: For large regions, consider processing in sub-regions and mosaicking results
- **Roughness normalization**: Handles varying Manning's coefficients across land covers (min–max scaling 0–1)
- **Slope percentile**: P95 slope used for normalization to avoid outlier-driven scaling
